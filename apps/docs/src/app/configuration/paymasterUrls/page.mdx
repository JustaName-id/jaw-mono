# paymasterUrls

Custom paymaster service URLs for sponsoring gas fees on different blockchain networks.

**Type:** `Record<number, string>`
**Required:** No
**Default:** `{}` (uses JAW default paymasters)

---

## Usage

```typescript
const jaw = JAW.create({
  apiKey: 'your-api-key',
  paymasterUrls: {
    1: 'https://paymaster.example.com/mainnet',
    137: 'https://paymaster.example.com/polygon',
    8453: 'https://paymaster.example.com/base',
  },
});
```

---

## Description

Paymasters are services that sponsor transaction gas fees, enabling gasless transactions for your users. The `paymasterUrls` configuration allows you to specify custom paymaster endpoints for each supported blockchain network.

**Benefits:**
- **Gasless Transactions**: Users don't need native tokens to pay gas
- **Better UX**: Remove friction from onboarding
- **Flexible Sponsorship**: Control which transactions to sponsor
- **Cost Management**: Use your own paymaster infrastructure

**Compliance:** Paymasters must follow [EIP-7677](https://eips.ethereum.org/EIPS/eip-7677) standard.

---

## Configuration Format

```typescript
paymasterUrls: {
  [chainId: number]: string // Paymaster URL
}
```

The key is the chain ID (decimal number), and the value is the paymaster service URL.

```typescript
const jaw = JAW.create({
  apiKey: 'your-api-key',
  paymasterUrls: {
    1: 'https://pm.example.com/eth',      // Ethereum Mainnet
    10: 'https://pm.example.com/op',      // Optimism
    137: 'https://pm.example.com/polygon', // Polygon
    8453: 'https://pm.example.com/base',  // Base
    42161: 'https://pm.example.com/arb',  // Arbitrum
  },
});
```

## How Paymasters Work

1. **User Initiates Transaction**: User sends a transaction request
2. **SDK Requests Sponsorship**: SDK calls paymaster with transaction details
3. **Paymaster Evaluates**: Paymaster decides whether to sponsor
4. **Paymaster Signs**: If approved, paymaster returns signature
5. **Transaction Executes**: Transaction executes with sponsored gas

```typescript
// User sends transaction
await jaw.provider.request({
  method: 'eth_sendTransaction',
  params: [{
    to: '0x...',
    value: '0x0',
    data: '0x...',
  }],
});

// Behind the scenes:
// 1. SDK contacts paymaster at paymasterUrls[chainId]
// 2. Paymaster evaluates and signs
// 3. Transaction executes with sponsored gas
```

## Sponsorship Strategies

### Sponsor All Transactions

```typescript
// Basic setup - sponsor everything
const jaw = JAW.create({
  apiKey: 'your-api-key',
  paymasterUrls: {
    137: 'https://paymaster.example.com/polygon',
  },
});
```

Your paymaster service can implement logic to sponsor all transactions from your app.

### Conditional Sponsorship

Implement conditions in your paymaster service:

```typescript
// Paymaster endpoint logic (example)
app.post('/paymaster', async (req, res) => {
  const { userOperation } = req.body;

  // Sponsor only if:
  // - Transaction value is below $10
  // - User has completed onboarding
  // - Daily limit not exceeded

  if (shouldSponsor(userOperation)) {
    const signature = await signUserOperation(userOperation);
    res.json({ signature });
  } else {
    res.status(402).json({ error: 'Not eligible for sponsorship' });
  }
});
```

### Chain-Specific Policies

```typescript
const jaw = JAW.create({
  apiKey: 'your-api-key',
  paymasterUrls: {
    // Sponsor all transactions on Polygon (cheap gas)
    137: 'https://paymaster.example.com/polygon/sponsor-all',

    // Sponsor only specific transactions on Ethereum (expensive gas)
    1: 'https://paymaster.example.com/mainnet/sponsor-limited',
  },
});
```

## Cost Management

### Budget Limits

Implement budget controls in your paymaster:

```typescript
// Paymaster service with daily budget
class PaymasterService {
  private dailyBudget = 1000; // $1000 per day
  private spentToday = 0;

  async sponsorTransaction(userOp) {
    const estimatedCost = await this.estimateCost(userOp);

    if (this.spentToday + estimatedCost > this.dailyBudget) {
      throw new Error('Daily budget exceeded');
    }

    this.spentToday += estimatedCost;
    return this.signUserOperation(userOp);
  }
}
```

### Per-User Limits

```typescript
// Limit sponsorship per user
class PaymasterService {
  private userLimits = new Map<string, number>();

  async sponsorTransaction(userOp) {
    const userAddress = userOp.sender;
    const userSpent = this.userLimits.get(userAddress) || 0;

    if (userSpent > 100) { // $100 per user
      throw new Error('User limit exceeded');
    }

    const cost = await this.estimateCost(userOp);
    this.userLimits.set(userAddress, userSpent + cost);

    return this.signUserOperation(userOp);
  }
}
```

## Multi-Paymaster Setup

### Primary and Fallback Paymasters

Your paymaster service can implement fallback logic:

```typescript
// Paymaster endpoint with fallback
app.post('/paymaster/primary', async (req, res) => {
  try {
    // Try primary sponsor (e.g., your budget)
    const signature = await primarySponsor(req.body);
    res.json({ signature });
  } catch (error) {
    // Fallback to secondary sponsor or reject
    const signature = await fallbackSponsor(req.body);
    res.json({ signature });
  }
});

const jaw = JAW.create({
  apiKey: 'your-api-key',
  paymasterUrls: {
    137: 'https://paymaster.example.com/primary',
  },
});
```

## Security Considerations

### Validate Requests

```typescript
// Verify requests come from your app
class PaymasterService {
  async sponsorTransaction(userOp, apiKey) {
    // Validate API key
    if (!this.validateApiKey(apiKey)) {
      throw new Error('Invalid API key');
    }

    // Validate user operation structure
    if (!this.validateUserOp(userOp)) {
      throw new Error('Invalid user operation');
    }

    return this.signUserOperation(userOp);
  }
}
```

### Rate Limiting

```typescript
// Prevent abuse with rate limiting
class PaymasterService {
  private rateLimits = new Map<string, number[]>();

  async sponsorTransaction(userOp) {
    const userAddress = userOp.sender;
    const now = Date.now();

    // Allow max 10 transactions per minute
    const recentTxs = this.rateLimits.get(userAddress) || [];
    const recentCount = recentTxs.filter(ts => now - ts < 60000).length;

    if (recentCount >= 10) {
      throw new Error('Rate limit exceeded');
    }

    recentTxs.push(now);
    this.rateLimits.set(userAddress, recentTxs);

    return this.signUserOperation(userOp);
  }
}
```

## Common Issues

### Paymaster Not Responding

```typescript
// ❌ Problem - Wrong URL or service down
const jaw = JAW.create({
  apiKey: 'your-api-key',
  paymasterUrls: {
    137: 'https://wrong-url.com/paymaster',
  },
});
```

Solution: Verify paymaster URL is correct and service is running.

### Transactions Not Sponsored

Even with paymasters configured, transactions may not be sponsored if:
- Paymaster budget exceeded
- User not eligible (per paymaster logic)
- Transaction doesn't meet sponsorship criteria

---

## Fallback Behavior

If paymaster sponsorship fails, transactions fall back to user-paid gas:

```typescript
const jaw = JAW.create({
  apiKey: 'your-api-key',
  paymasterUrls: {
    137: 'https://paymaster.example.com/polygon',
  },
});

// If paymaster denies sponsorship:
// 1. SDK attempts paymaster sponsorship
// 2. Paymaster rejects (budget, conditions, etc.)
// 3. User pays gas with their own funds
// 4. Transaction proceeds normally
```
