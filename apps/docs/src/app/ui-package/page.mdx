# @jaw/ui Package

Pre-built React UI handler for app-specific mode. This package provides a `ReactUIHandler` that handles all wallet approval dialogs automatically when used with the core `JAW.create()` function.

---

## Installation

```bash
npm install @jaw/ui @jaw.id/core
# or
yarn add @jaw/ui @jaw.id/core
# or
bun add @jaw/ui @jaw.id/core
```

---

## ReactUIHandler

A UIHandler implementation for React that renders built-in UI dialogs for wallet operations.

### Usage

```typescript
import { JAW, Mode } from '@jaw.id/core';
import { ReactUIHandler } from '@jaw/ui';

// Create JAW SDK with app-specific mode
const jaw = JAW.create({
  apiKey: 'your-api-key',
  defaultChainId: 1,
  paymasterUrls: {
    1: 'https://paymaster.example.com/ethereum',
    8453: 'https://paymaster.example.com/base',
  },
  preference: {
    mode: Mode.AppSpecific,
    uiHandler: new ReactUIHandler(),
  },
});

// Use the provider
const result = await jaw.provider.request({
  method: 'wallet_connect',
  params: [{ version: '1.0' }],
});
```

The `ReactUIHandler` is automatically initialized by the SDK with the necessary configuration (`apiKey`, `defaultChainId`, `paymasterUrls`). You don't need to pass any parameters to the constructor - the SDK handles this for you via the `init()` method.

---

## Built-in Dialogs

ReactUIHandler automatically renders appropriate dialogs for each wallet operation:

| Dialog | Triggered By | Description |
|--------|--------------|-------------|
| **OnboardingDialog** | `wallet_connect` (new user) | Account creation with passkey |
| **ConnectDialog** | `wallet_connect` (existing user) | Connection confirmation |
| **SignatureDialog** | `personal_sign` | EIP-191 message signing |
| **SiweDialog** | `personal_sign` (SIWE message) | Sign-In with Ethereum |
| **Eip712Dialog** | `eth_signTypedData_v4` | EIP-712 typed data signing |
| **TransactionDialog** | `wallet_sendCalls`, `eth_sendTransaction` | Transaction approval |
| **PermissionDialog** | `wallet_grantPermissions`, `wallet_revokePermissions` | Permission management |

Features:
- **TransactionDialog**: Automatic gas estimation with paymaster sponsorship indication
- **All dialogs**: Address resolution (ENS/subnames), mobile-responsive design

## Complete Example

```typescript
'use client';

import { useState, useEffect } from 'react';
import { JAW, Mode, ProviderInterface } from '@jaw.id/core';
import { ReactUIHandler } from '@jaw/ui';

function WalletApp({ provider }: { provider: ProviderInterface }) {
  const [address, setAddress] = useState<string | null>(null);

  const connect = async () => {
    try {
      const result = await provider.request({
        method: 'wallet_connect',
        params: [{ version: '1.0' }],
      });
      setAddress(result.accounts[0].address);
    } catch (error) {
      console.error('Connection failed:', error);
    }
  };

  const signMessage = async () => {
    if (!address) return;

    const message = 'Hello from my dApp!';
    const hexMessage = '0x' + Buffer.from(message).toString('hex');

    const signature = await provider.request({
      method: 'personal_sign',
      params: [hexMessage, address],
    });

    console.log('Signature:', signature);
  };

  return (
    <div>
      {address ? (
        <>
          <p>Connected: {address}</p>
          <button onClick={signMessage}>Sign Message</button>
        </>
      ) : (
        <button onClick={connect}>Connect Wallet</button>
      )}
    </div>
  );
}

export default function App() {
  const [provider, setProvider] = useState<ProviderInterface | null>(null);

  useEffect(() => {
    const jaw = JAW.create({
      apiKey: process.env.NEXT_PUBLIC_JAW_API_KEY!,
      defaultChainId: 8453,
      paymasterUrls: {
        8453: 'https://paymaster.example.com/base',
      },
      preference: {
        mode: Mode.AppSpecific,
        uiHandler: new ReactUIHandler(),
      },
    });

    setProvider(jaw.provider);

    return () => {
      jaw.disconnect();
    };
  }, []);

  if (!provider) return <div>Loading...</div>;

  return <WalletApp provider={provider} />;
}
```
