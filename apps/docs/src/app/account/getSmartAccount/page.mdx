# account.getSmartAccount()

Get the underlying viem SmartAccount for advanced operations.

**Type:** `instance`

---

## Signature

```typescript
getSmartAccount(): SmartAccount
```

---

## Parameters

None.

---

## Returns

`SmartAccount` - The raw viem SmartAccount instance.

---

## Behavior

Returns the underlying viem `SmartAccount` instance, giving you direct access to all viem smart account methods and properties.

---

## Examples

### Basic Usage

```typescript
import { Account } from '@jaw.id/core';

const account = await Account.get({
  chainId: 1,
  apiKey: 'your-api-key',
});

const smartAccount = account.getSmartAccount();
console.log('SmartAccount type:', smartAccount.type);
```

### Access Factory Address

```typescript
const smartAccount = account.getSmartAccount();

// Access account abstraction properties
console.log('Address:', smartAccount.address);
```

### Use with viem Directly

```typescript
import { Account } from '@jaw.id/core';
import { createPublicClient, http } from 'viem';
import { mainnet } from 'viem/chains';

const account = await Account.get({
  chainId: 1,
  apiKey: 'your-api-key',
});

const smartAccount = account.getSmartAccount();

// Create a public client
const publicClient = createPublicClient({
  chain: mainnet,
  transport: http(),
});

// Use viem methods directly
const nonce = await smartAccount.getNonce();
console.log('Current nonce:', nonce);
```

### Custom User Operation

```typescript
import { Account } from '@jaw.id/core';

const account = await Account.get({
  chainId: 1,
  apiKey: 'your-api-key',
});

const smartAccount = account.getSmartAccount();

// Sign a user operation directly
const signature = await smartAccount.signUserOperation({
  callData: '0x...',
  callGasLimit: 100000n,
  // ... other user op fields
});
```

---

## When to Use

Use `getSmartAccount()` when you need:

- Direct access to viem smart account methods
- Custom user operation handling
- Integration with other viem-based tools
- Advanced account abstraction features

For most use cases, the `Account` class methods (`sendTransaction`, `signMessage`, etc.) are sufficient.
